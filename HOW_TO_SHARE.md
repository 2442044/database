# 共有データベース・システム起動マニュアル

このシステムを複数のPCから利用したり、別のPCで同じデータを引き継いで起動したりする方法です。

## 1. 別のPCで「同じデータ」を使って起動する方法
GitHubにデータベースファイル (`dvd_rental.db`) を含めています。

1.  **最新のデータを取得する**
    ```bash
    git pull origin main
    ```
2.  **システムを起動する**
    ```bash
    python app.py
    ```
    これにより、GitHubにアップロードされている最新の状態で起動します。
    ※データを更新した後は、`git add dvd_rental.db`, `git commit`, `git push` を行うことでGitHub上のデータが更新されます。

## 2. 1人が起動したシステムに他のPCからアクセスする方法（リアルタイム共有）
同じWi-Fiやネットワーク内にいる場合、1台のPCで動いているシステムに全員でアクセスできます。

1.  **ホストPC（システムを起動する人）**
    - `python app.py` で起動します。
    - 自分のIPアドレスを確認します（例: `192.168.xx.xx`）。
2.  **他のPC（アクセスする人）**
    - ブラウザを開き、ホストPCのIPアドレスとポート番号を入力します。
    - 例: `http://192.168.xx.xx:5000`

## 注意事項
- **SQLiteの制限**: 複数の人が同時に書き込みを行うと、データベースがロックされることがあります。
- **Gitのコンフリクト**: 複数の人が同時に `dvd_rental.db` を書き換えてプッシュするとコンフリクトが発生します。データの同期は、一人が更新・プッシュした後に他の人がプルする形で行ってください。
